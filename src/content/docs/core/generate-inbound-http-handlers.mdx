The einar-cli allows you to generate custom handlers with the necessary files and imports to handle your request. After generating the handler, you must manually replace the placeholder `/insert-your-custom-pattern-here` with the actual path you intend to use in your application.

## üë®‚Äçüíª Generate a New Custom Handler

Inside your project directory, run the following command to create a new custom handler:
<Tabs>

  <TabItem label="POST">
```sh
einar generate post-controller post-customer
```

Here's an example of how the generated code might look:
```sh
func init() {
	ioc.Registry(postCustomer, serverwrapper.NewEchoWrapper)
}
func postCustomer(e serverwrapper.EchoWrapper) {
	e.POST("/insert-your-custom-pattern-here", func(c echo.Context) error {
		return c.JSON(http.StatusOK, map[string]string{
			"message": "Unimplemented",
		})
	})
}
```
The file `post_customer.go` will be created in the following directory structure:
```
/app
  /adapter
    /in
      /api
        - post_customer.go  
```

## üåê Check The Handler Response
Execute the following command to retrieve the default response of your generated handler:

<Tabs>
  <TabItem label="CURL">
```sh
curl -X POST http://localhost:8080/insert-your-custom-pattern-here
```
  </TabItem>

  <TabItem label="Windows PowerShell">
```sh
Invoke-WebRequest -Uri http://localhost:8080/insert-your-custom-pattern-here -Method POST
```
  </TabItem>
</Tabs>

ResponseBody:
```sh
{
    "message": "Unimplemented"
}
```
  </TabItem>
